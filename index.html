The Pros and Cons of unit testing

Pros

  • Simple
  • Fast to write
  • Easy to learn and setup

Cons

  • Require lots of code
  • Can be laborious to refactor
  • Requires lots of creativity

Example Time

  • Performonce rewrite
  • Good test coverage

The Side Effects

  • Passed unit tests
  • Passed integration tests
  • Accepted after human tests
  • Failing only visible when other features written

What Could We Do?

  • Not depend on human creativity
  • Let machines deal with randomness
  • Test more cases!

Generative Example


(defspec sort-check 100
  (for-all [v (gen/not-empty (gen/vector gen/int))]
           (let [models (create-models v)
                 sorted (sort v)
                 subject (tracker.PanelItems.)]
             (.reset subject (apply array models))
             (.refresh subject (apply array (sort-by #(.get % :id) models)))
             (= sorted
                (ids subject)))))
